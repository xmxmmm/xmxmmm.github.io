之前用桥接模式时 ping 不通 Gitee，大概率是 物理网络环境对 “新设备”（虚拟机）有隐性限制，比如：

物理路由器的 DHCP 服务虽然给了虚拟机 IP，但限制了该 IP 的外网访问权限；

物理网络的 DNS 服务器（比如公司 / 校园网的内网 DNS）无法解析 Gitee 的域名；

物理网络的防火墙规则拦截了虚拟机 IP 的出站请求（比如禁止非备案设备访问外网）。

而切换到 NAT 模式后，虚拟机的所有网络流量都会伪装成宿主机的流量（通过宿主机的 IP 和网络配置转发）—— 既然你的物理机本身能正常访问 Gitee，虚拟机自然也能 “借道” 通了。

暂停美化，解决新问题

我在NAT模式下怎么才能用ssh连接到虚拟机呢？

在 NAT 模式下，由于虚拟机的 IP 是 “私有网段”（仅能被宿主机识别，无法被局域网其他设备直接访问），要通过 SSH 连接虚拟机，需要通过 “端口转发”（Port Forwarding）将宿主机的端口映射到虚拟机的 SSH 端口（默认 22）。

VirtualBox

关闭虚拟机，右键点击虚拟机 →「设置」→「网络」；

选择「NAT」模式，点击「高级」→「端口转发」；

点击「+」添加规则：

名称：可选（如 “SSH”）；
协议：TCP；
主机 IP：留空（或填 127.0.0.1 仅允许本地连接）；
主机端口：2222（同上，自定义端口）；
子系统 IP：虚拟机 IP，在虚拟机里ip addr查看；
子系统端口：22；

保存设置，启动虚拟机。

其他设备的终端执行 ssh 虚拟机用户名@宿主机局域网IP -p 2222

如果我只在宿主机上ssh虚拟机

ssh ubuntu@localhost -p 2222 就ok

安装zsh -autosuggestions 插件

zsh-autosuggestions 是一个命令提示插件，当你输入命令时，会自动推测你可能需要输入的命令，按下右键可以快速采用建议。

git clone https://gh.xmly.dev/https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

zsh-syntax-highlighting 插件

zsh-syntax-highlighting 是一个命令语法校验插件，在输入命令的过程中，若指令不合法，则指令显示为红色，若指令合法就会显示为绿色。

git clone https://gh.xmly.dev/https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

启用插件

vim ~/.zshrc

再装一个fzf，fzf 是一款命令行模糊搜索工具，能与 shell 深度集成，Ctrl+R 进行搜索

sudo apt install fzf

还是把它放到vim ~/.zshrc的插件列表里就可以了
